# 19｜企业日常任务手册

本节不是知识点列表，而是值班时可照着走的最小操作流程。所有变更先确认环境、集群、索引和审批范围；示例中的占位符必须替换后再执行。

## 任务 1：每日健康巡检

```http
GET /_cluster/health
GET /_cat/nodes?v&h=name,ip,roles,master,heap.percent,cpu,load_1m,disk.used_percent
GET /_cat/indices?v&h=health,status,index,pri,rep,docs.count,store.size&s=store.size:desc
GET /_cat/shards?v&h=index,shard,prirep,state,unassigned.reason,node&s=state
GET /_cluster/pending_tasks
GET /_slm/status
```

验收：集群状态符合架构预期；没有意外的未分配分片；磁盘、堆内存和处理器没有持续异常趋势；快照成功；文档增长与业务量一致。发现偏差要记录时间、节点和索引，不能只截一张绿色状态图。

## 任务 2：新建业务索引

1. 收集负责人、用途、数据量与增长速度、保留期、查询、更新、权限、恢复点目标和恢复时间目标。
2. 评审映射、主分片与副本、模板优先级和命名。
3. 创建组件模板与索引模板，并先进行模拟。
4. 创建版本化索引，例如 `orders-v1-000001`。
5. 创建读/写别名，授予最小权限。
6. 写入样本，验证查询、聚合、错误输入和监控。
7. 登记操作手册、仪表盘、告警和快照范围。

## 任务 3：映射变更与重新索引

1. 新建 `products-v2`，绝不在原索引上尝试修改不兼容的字段类型。
2. 记录旧索引的文档数量，启动异步重新索引任务。
3. 监控任务、目标索引速率、拒绝次数、磁盘和业务影响。
4. 处理迁移期间增量：双写、变更日志追平或短暂停写，选一种并验证。
5. 比较文档数、抽样哈希值或字段、关键查询和相关性。
6. 原子切换别名；观察错误率和业务指标。
7. 出错则别名切回；旧索引保留到回滚窗口结束，再按审批清理。

```http
POST /_aliases
{
  "actions": [
    {"remove":{"index":"products-v1","alias":"products-read"}},
    {"add":{"index":"products-v2","alias":"products-read"}}
  ]
}
```

## 任务 4：清理历史数据

优先通过索引生命周期管理或数据流生命周期按保留策略自动删除。临时清理前：

- 确认数据负责人、合规保留期、查询依赖和快照。
- 精确列出索引，不用宽泛通配符。
- 先执行 `GET /_cat/indices/<pattern>?v` 复核目标和数据量。
- 在变更窗口操作并观察集群恢复与磁盘。

按查询执行大规模 `_delete_by_query` 会产生读取、写入和段合并压力；对于时间序列数据，删除整个索引通常更高效。

## 任务 5：处理磁盘告警

1. 查看节点磁盘和最大索引。
2. 检查数据增长、索引生命周期管理状态、快照和分片分布。
3. 扩盘/加节点是优先长期方案；合规允许时删除已过期且有保护的数据。
4. 观察分片迁移、恢复和写入是否恢复正常。
5. 修正容量预测、保留策略或滚动更新条件。

禁止执行的动作：删除不了解用途的索引、删除节点数据目录、长期提高磁盘水位阈值来掩盖问题。

## 任务 6：处理集群红色状态

```http
GET /_cluster/health?level=indices
GET /_cat/shards?v&s=state
POST /_cluster/allocation/explain
{}
```

先判断是节点暂时离线、磁盘、配置还是主分片数据不可用。恢复节点或从快照恢复优先。接受陈旧主分片、空主分片或丢数据操作必须明确损失范围并获得授权。

## 任务 7：慢查询工单

向提单方索取：完整时间范围、请求标识、索引或别名、脱敏后的查询 DSL、第 50/95/99 百分位延迟、期望的服务等级目标，以及近期是否发布过新版本。检查以下内容：

1. 应用端到端延迟与 Elasticsearch `took` 的差异。
2. 当时的处理器、堆内存与垃圾回收、磁盘、拒绝次数和并发情况。
3. 查询涉及的分片数、返回量、分页、聚合、脚本和通配符。
4. 映射、分析器、字段基数和数据倾斜。
5. 在测试环境分析代表性请求，每次只修改一个变量。

## 任务 8：创建应用权限

1. 确认读取、写入、删除、索引范围、环境、负责人和有效期。
2. 创建最小权限角色或受限的 API 密钥。
3. 用 `_has_privileges` 验证允许与拒绝路径。
4. 通过密钥系统交付，不在工单中粘贴完整凭据。
5. 登记到期、轮换、吊销和审计联系人。

## 任务 9：节点维护

1. 确认集群健康、快照和维护窗口。
2. 按官方维护流程控制分片分配，避免节点短暂重启触发大量复制。
3. 一次只处理计划允许的节点，停止服务前确认节点身份。
4. 启动后等待节点加入，验证日志、分片恢复和健康状态。
5. 恢复临时配置，观察业务服务等级目标后关闭变更。

不同版本和节点角色的滚动顺序不同，升级操作必须使用目标版本官方指南，不能只套用普通重启流程。

## 任务 10：交接与复盘

交接需包含当前状态、用户影响、已执行命令及响应摘要、未完成步骤、风险、下一决策点、监控链接和负责人。所有临时设置必须登记到期时间，避免“临时永久化”。

## 值班验收

你应能在不依赖搜索引擎临时拼接命令的情况下，安全完成巡检、创建索引、别名迁移、权限申请、磁盘告警初判、黄色或红色状态初判，以及慢查询信息收集。
